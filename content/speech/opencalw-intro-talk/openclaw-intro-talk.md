## 一、引子：信息过载，你的时间去哪了？（3 分钟）

**目标：用信息过载的共鸣场景切入，建立"现在 vs 以后"的对比**

**Before（现状痛点）**：

- 每天有读不完的文章、文档、POPO群消息
- 信息碎片化，看了就忘，没有沉淀
- 需要花大量时间手动整理、汇总、分发

**After（OpenClaw 接管后）**：

- 资讯自动聚合提炼，早报 3 分钟搞定
- 内容自动处理入库，知识真正被沉淀
- 重复性信息流转交给它，你只做决策

**过渡句**：大多数人用 OpenClaw 只是在"聊天"——今天我们聊聊怎么让它真正"干活"。

---

## 二、什么时候应该用 OpenClaw？（5 分钟）

**目标：帮听众建立判断力，知道哪些场景值得投入**

OpenClaw 最擅长这类任务：

| 特征                | 举例                                             |
| ------------------- | ------------------------------------------------ |
| **重复性高**        | 每天写日报、整理 Issue、刷资讯                   |
| **信息密集**        | 读长文档、做技术选型调研                         |
| **有明确输入/输出** | "把这篇英文文章翻译+提炼"、"给我今日 Issue 列表" |
| **不需要人类判断**  | 标签分类、格式整理、内容聚合                     |

反过来——**这些事情不适合交给它**：

- 需要你的经验和直觉做决策
- 涉及高风险操作（发布代码）——至少不能让它独立完成

## 三、如何正确使用 OpenClaw？（8 分钟）

**目标：Prompt 设计思路 + 实际内容演示，非技术同学也能理解**

### 3.1 核心思路：把它当成一个新来的聪明实习生

- 它什么都会，但不了解你、不了解你的项目、不了解你的偏好
- 你需要告诉它：**你是谁、你的习惯、它能做什么、不能做什么**
- 配置文件（`SOUL.md` / `USER.md`）就是这份"入职材料"，一次配置，长期受益

### 3.2 写好 Prompt 的三个原则

1. **明确输入**：告诉它从哪里取数据（链接、文件路径、POPO群）
2. **明确输出**：告诉它结果是什么格式、存到哪里、推送给谁
3. **划清红线**：高风险操作（推送消息、修改文件）必须等你确认再执行

### 3.3 现场演示：一个完整 Prompt 的拆解

> 演示内容：整理今日POPO群消息 → 按话题分类 → 生成待跟进列表

```
读取今天POPO群「产品反馈」里的消息，完成以下处理：

1. 按话题分类（Bug 反馈 / 功能建议 / 使用咨询）
2. 每类列出 3 条最高频的问题
3. 生成一份「本日待跟进清单」，按优先级排序
4. 发送到我的POPO个人会话，等待我确认后再推送到群

```

**重点拆解**：输入在哪 → 输出是什么 → 推给谁 → 高风险操作等确认

### 3.4 纠错闭环：犯过的错不能犯两次

- 问题：AI 会反复犯同一类错误（猜参数、不确认就执行）
- 解法：每次纠错后，让它把教训**用一句话写下来存好**，下次会话开始自动回顾
- 类比：就像给新同事建一个「踩坑备忘录」，积累得越多，它越少出错
- 这是从「偶尔好用」变成「越用越顺手」的关键

## 四、OpenClaw 实战（17 分钟）

### 场景 A：知识管理（6 分钟，适合所有人）

**痛点**：信息过载，内容看了就忘，知识沉淀效率低

- **内容处理**：英文长文 → 一句话摘要 + 中文要点提炼 + 自动入库
- **资讯聚合**：读取多个信源（技术博客、POPO群），生成今日精选早报
- **晨间摘要**：自动读取今日任务，整理昨日完成 + 今日 Top 3 优先级

> 演示 Prompt：
>
> ```
> 帮我处理这篇文章：[URL]
>
> 1. 用一句话说清楚核心观点
> 2. 按原文结构，用中文提炼每节要点（每节不超过 3 句）
> 3. 找出最有价值的 3 段原文，翻译为流畅中文
> 4. 整理为 Markdown 笔记，存入知识库
> ```
>
> 演示重点：展示完整输出结果，对比"自己读"需要的时间

---

### 场景 B：项目管理（6 分钟，适合 PM、TL）

**痛点**：反馈分散、Issue 混乱、进展同步靠人肉

- **反馈收集与整理**：抓取用户反馈，自动分类汇总（Bug / 功能建议 / 体验问题），生成结构化报告
- **Issue 智能分诊**：新 Issue 自动打标签、判断优先级、指派负责人；高优问题直推告警群
- **进展推送**：抓取 GitLab 活动，生成日报草稿，等你确认后推送到项目群

> 演示重点：POPO反馈 → 分类汇总报告的 Prompt 示例（建议提前准备一段POPO群截图作为输入素材）

---

### 场景 C：研发辅助（5 分钟，适合工程师）

**痛点**：技术调研耗时、新仓库上手慢、英文文档阅读效率低

- **技术调研**：给一个技术主题，自动搜集资料、生成多方案对比报告（成熟度 / 性能 / 生态 / 学习曲线）
- **代码阅读**：系统性分析陌生仓库的架构、入口、核心抽象，快速建立认知地图
- **知识整理**：官方文档 / API 变更说明 → 中文提炼 + 关键影响点标注，直接存入知识库

> 演示重点：给一个 GitHub 仓库链接 → 输出架构概览的 Prompt 示例

---

## 五、行动建议（2 分钟）

**给听众一个「今天就能做」的第一步**

1. **今天**：用一个具体任务试运行一次，感受 Prompt 输入 → 输出的完整流程
2. **这周**：把你最重复的一件事交给它跑通，比如日报或资讯聚合
3. **长期**：它犯错时让它把教训记下来，积累越多它越懂你
